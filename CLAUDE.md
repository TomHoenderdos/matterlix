# Matterlix

Elixir NIF bindings for the Matter (CHIP) SDK. This is a **reusable library** — firmware projects depend on it via `{:matterlix, "~> 0.3"}`.

## Project Structure

```
matterlix/
├── c_src/
│   └── matter_nif.cpp           # C++ NIF (Matter SDK bindings)
├── lib/
│   ├── matterlix.ex             # Convenience API (update_attribute, get_attribute)
│   └── matterlix/
│       ├── application.ex       # OTP Application (auto_supervise config)
│       ├── matter.ex            # GenServer API (high-level)
│       ├── handler.ex           # Handler behaviour for event dispatch
│       ├── handler/default.ex   # Default no-op handler
│       ├── device_profiles.ex   # Device profile definitions
│       └── matter/
│           └── nif.ex           # Low-level NIF bindings
├── lib/mix/tasks/
│   └── matterlix.build_sdk.ex   # Mix task for Docker-based SDK build
├── Makefile                     # NIF build (stub mode + SDK mode)
├── matter_sdk_includes.mk       # Auto-generated SDK link config
├── docker/                      # Docker build for arm64 SDK
├── scripts/                     # SDK build & include generation scripts
├── example/                     # Example firmware project
└── config/
    ├── config.exs
    └── host.exs
```

## Key Architecture

- **Library deps**: only `elixir_make` + `ex_doc`
- **Stub mode** (default): NIF compiles without Matter SDK, returns stub values
- **SDK mode** (`MATTER_SDK_ENABLED=1`): Links against libCHIP.a + ~111 .o files + 43 static libs
- **Handler behaviour**: `Matterlix.Handler` — consuming projects implement callbacks
- **Device profiles**: `Matterlix.DeviceProfiles` — light, lock, sensor, thermostat, etc.
- **Convenience API**: `Matterlix.update_attribute/4`, `Matterlix.get_attribute/3`

## Build System

- `Makefile` uses `MATTER_SDK_ENABLED`, `MATTER_DEBUG`, `ASAN` flags
- `matter_sdk_includes.mk` is auto-generated by `scripts/gen_matter_includes.sh`
- Cross-compilation: consuming firmware passes `CROSSCOMPILE=1` + `MATTER_SDK_ENABLED=1` via make_env
- Static lib link order matters: `-lCHIP` and .o files MUST come AFTER object files
- 43 static libraries use `-Wl,--start-group ... -Wl,--end-group` for circular deps

## Quick Start

```bash
mix deps.get && mix compile   # stub mode
mix test                      # 55 tests, 0 failures
```
